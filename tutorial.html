01

Snap-Solver 是什么？

Snap-Solver 是一个在你自己电脑上运行的截屏解题工具。它的工作方式是「手机遥控电脑」：

  • 电脑上运行程序，手机浏览器打开页面；
  • 手机上点一下 截屏解题，电脑自动截取它的整个屏幕并回传手机；
  • 在手机上缩放、框选题目区域，交给你选择的 AI 模型；
  • AI 流式给出解答——推理模型的思考过程实时可见，答案支持公式和代码渲染；
  • 答案没看懂，还可以就这道题继续追问。

整个应用基于 Python + Flask，密钥和数据都只存在你的电脑上，不经过任何第三方服务器。

02

准备清单

  • 一台可以联网的 Windows、macOS 或 Linux 电脑，以及一部同一 Wi-Fi 下的手机；
  • 至少一个可用的模型 API Key（推荐准备 2~3 个，方便切换），六选一即可：Anthropic（Claude）、OpenAI（GPT）、Google（Gemini）、阿里通义（Qwen）、字节豆包、Moonshot（Kimi）；
  • 国内网络环境建议优先申请 阿里通义 / 豆包 / Kimi，无需代理即可直连；
  • 约 2 GB 可用硬盘空间。
提示：Snap-Solver 不依赖显卡或 GPU，普通轻薄本即可顺利运行。
03

第一次打开命令行

Snap-Solver 需要在命令行里执行几条简单的指令。命令行是一个黑色（或白色）窗口，通过输入文字来让电脑完成任务。不同系统打开方式略有区别：

Windows

  1. 按 Win + S，输入 cmd 或 terminal，打开 命令提示符 或 Windows Terminal；
  2. 复制命令后在窗口里右键 → 粘贴，或 Ctrl + V；
  3. 切换到某个文件夹（例如 D:\Snap-Solver）：
cd /d D:\Snap-Solver

macOS

  1. 按 Command + Space 呼出 Spotlight，输入 Terminal 回车；
  2. 切换到项目目录（例如在「下载」文件夹内）：
cd ~/Downloads/Snap-Solver

Linux（Ubuntu 示例）

cd ~/Snap-Solver
常用命令速记：cd 路径 进入文件夹（路径有空格用双引号包住）；dir（Windows）/ ls（macOS、Linux）查看当前文件夹内容；按方向键 ↑ 调出上一条命令。
04

安装 Python 3

Snap-Solver 基于 Python 3.9+，推荐 3.10 或 3.11。

Windows

  1. 访问 python.org/downloads，下载最新稳定版的 Windows 安装包；
  2. 双击安装，务必勾选 Add Python to PATH；
  3. 装完后打开命令行验证：
python --version
pip --version

macOS

从 python.org 下载 .pkg 安装包按提示安装，然后在终端验证（macOS 下命令用 python3 / pip3）：

python3 --version
pip3 --version

Linux（Ubuntu 示例）

sudo apt update
sudo apt install python3 python3-venv python3-pip -y

06

获取 Snap-Solver 项目代码

任选其一：

方式一：Git 克隆（推荐）

git clone https://github.com/Zippland/Snap-Solver.git
cd Snap-Solver

方式二：下载压缩包

  1. 打开 项目主页，点 Release → Source code (zip)；
  2. 解压后把文件夹重命名为 Snap-Solver 并记住路径。

后续步骤默认你已位于项目根目录（包含 app.py、requirements.txt 的文件夹）。

07

创建虚拟环境并安装依赖

虚拟环境可以把项目依赖和系统环境隔离，避免冲突。

创建并激活虚拟环境

Windows PowerShell：

python -m venv .venv
.\.venv\Scripts\Activate

macOS / Linux：

python3 -m venv .venv
source .venv/bin/activate

激活成功后，命令行前面会出现 (.venv) 前缀。关掉窗口后需要重新进入项目目录并再次激活。

安装依赖

pip install --upgrade pip
pip install -r requirements.txt
如果安装超时：可以临时用清华镜像源 pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt。
08

首次启动与访问

  1. 确保虚拟环境处于激活状态；
  2. 在项目根目录执行：
python app.py
→ Running on http://192.168.1.8:5000
  1. 终端出现 Running on http://...:5000 即启动成功（5000 被占用时会自动改用 5001，以终端打印的地址为准）；
  2. 手机与电脑连同一 Wi-Fi，手机浏览器输入终端里打印的地址即可打开。电脑重连网络后 IP 可能变化，连不上时回终端重新确认。
暂停服务：在终端按 Ctrl + C。再次启动只需重新激活虚拟环境并执行 python app.py。
09

配置 API 密钥与基础设置

首次打开：跟着引导走

第一次在手机上打开页面，会看到一段工作机制说明，点 「知道了，去选模型」 直接进入模型选择页：

  1. 顶部横排是六家厂商的标签（带小圆点的表示还没配密钥）；
  2. 选中你有密钥的厂商，页面顶部会出现密钥填写卡——粘贴密钥、点保存即可；
  3. 卡片右上角有 「如何获取」 链接，直达该厂商的密钥申请页面；
  4. 在「全部模型」列表里点选一个模型，选择即生效。

之后想换模型或换密钥：点首页底部的模型卡片随时进入这个页面；已配的密钥会沉底显示，点「修改」可以更换（清空后保存 = 删除该密钥）。

设置页一览

点右上角的齿轮进入设置，共三组：

解题提示词模式（内置多种，也可自建）、回复语言
模型接入API 密钥集中管理、网络（代理与中转）
通用外观（浅色/深色/跟随系统）、GitHub 仓库、关于与检查更新

所有密钥只保存在你电脑的 .snapsolver/api_keys.json 里（该目录已被 git 忽略，不会被误提交）。

设置代理与中转（可选）

在 设置 → 网络 中：

  • HTTP 代理：打开开关，填本地代理的主机和端口（如 127.0.0.1 / 7890），用于访问 Anthropic、OpenAI、Google 等国际 API；
  • 中转 API 地址：每家厂商单独一栏，填了即走中转，留空走官方地址；
  • 修改立即生效，无需重启应用。

如何确认 VPN / 代理端口

很多加速器或 VPN 客户端会在本地启动一个「系统代理」服务（常见端口如 7890、1080）。找端口的常见途径：

  • 打开 VPN 客户端设置，找「本地监听端口」「HTTP(S) 代理」「SOCKS 代理」等字样；
  • Windows：「设置 → 网络和 Internet → 代理」查看代理服务器地址和端口；
  • macOS：「系统设置 → 网络 → Wi-Fi → 详情 → 代理」查看勾选的服务和端口。

拿到端口后，在 Snap-Solver 的代理设置里填 127.0.0.1 和对应端口即可。

10

获取常用 API Key

API Key 相当于你在各大模型平台上的「门票」。申请过程中务必保护好账号安全，切勿向他人泄露密钥。

Anthropic（Claude 系列）

  1. 打开 console.anthropic.com 注册账号，完成手机验证与支付绑定；
  2. 进入 API Keys 页面 → Create Key；
  3. 复制生成的密钥（形如 sk-ant-...），在模型页选择 Anthropic 后粘贴保存。

OpenAI（GPT 系列）

  1. 打开 platform.openai.com 注册登录，完成支付方式绑定；
  2. 进入 API keys → Create new secret key；
  3. 复制密钥（形如 sk-...），在模型页选择 OpenAI 后粘贴保存。

Google Gemini

  1. 前往 ai.google.dev 登录 Google 账号；
  2. 点 Get API key，选择或创建项目生成密钥（形如 AIza...）；
  3. 在模型页选择 Google 后粘贴保存。

阿里通义（Qwen / QVQ，国内直连）

  1. 打开 dashscope.console.aliyun.com，用阿里云账号登录；
  2. 进入「API Key 管理」→ 创建 API Key；
  3. 复制密钥，在模型页选择通义后粘贴保存。收费模型需先实名认证并开通计费。

字节豆包（火山方舟，国内直连）

  1. 打开 console.volcengine.com/ark 注册登录并完成实名认证；
  2. 在「API Key 管理」中创建 API Key；
  3. 复制密钥，在模型页选择豆包后粘贴保存。

Moonshot Kimi（国内直连）

  1. 打开 platform.moonshot.cn 注册登录；
  2. 进入「API Key 管理」→ 新建；
  3. 复制密钥（形如 sk-...），在模型页选择 Kimi 后粘贴保存。
安全小贴士：API Key 和密码一样重要，泄露后他人可能代你调用接口、消耗额度。建议为不同用途创建多个密钥，定期检查和撤销不用的密钥；平台支持额度上限的可以酌情启用。
11

完成第一次题目解析

  1. 电脑上把题目显示在屏幕上（网页、PDF、课件都可以）；
  2. 手机页面左上角确认显示绿色的 「已连接 · 电脑端」；
  3. 手机上点大圆按钮 「截屏解题」——电脑自动截取整个屏幕并回传到手机；
  4. 在手机上双指缩放、拖动框选题目区域（右上角实时显示所选区域的像素尺寸），点 「发送解题」；
  5. 解答页会依次出现：思考过程（淡色小字实时展开，结束自动收起，点击可重新展开）→ 最终答案（Markdown 渲染，公式清晰、代码块可一键复制）。生成中可随时点右上角停止；
  6. 答案出来之后：
    • 没看懂？在底部输入框就这道题继续追问，AI 会带着题目图片和之前的问答继续回答（换题或返回后自动清空）；
    • 想换个角度？答案下方有 重解 / 换模型重解 / 复制 三个快捷操作；
    • 继续刷题：返回首页再点「截屏解题」，上一次的框选位置会自动保留。
小技巧：点解答页顶部的题目缩略图可以全屏对照原题；终端会实时输出请求日志，方便排查问题。
12

常见问题速查

  • python 命令找不到：Windows 重开终端或重启电脑，或改用 py；macOS / Linux 用 python3。
  • pip install 超时：临时用清华源 pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt。
  • 启动后网页打不开：确认终端没有报错；检查防火墙、端口占用（5000 被占用时自动改用 5001，以终端打印的地址为准），或试 http://127.0.0.1:5000。
  • 截图没反应 / 截出来是黑屏：macOS 需在「系统设置 → 隐私与安全性 → 屏幕录制」中勾选终端（或运行 Python 的应用）并重启程序；Windows 一般无需额外授权。
  • 模型报 401 / 403：检查 API Key 是否正确、账号余额是否充足，必要时换模型或配置中转地址。
  • 手机访问失败：确保手机和电脑在同一个 Wi-Fi 下，且电脑未开启导致局域网隔离的 VPN。
13

进一步探索

  • 自建提示词：设置 → 解题 → 提示词模式 → 新建，可为特定学科定制解题风格（存放在 .snapsolver/prompts.json）；
  • 推理档位：模型页当前模型卡上的 Fast / High / Max 三档，全局统一，复杂题目调高档位效果更好；
  • config/models.json：模型列表的配置文件，进阶用户可按需增删模型；
  • 更新项目：Git 克隆的执行 git pull；压缩包用户重新下载覆盖。应用内「设置 → 关于 → 检查更新」也会提示新版本。
